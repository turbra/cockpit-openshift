    Supported installation model
    This flow assumes the target host already has access to the Red Hat package repositories required for microshift and openshift-clients.
    The Red Hat installation model expects an existing RHEL host with package access, passwordless sudo, and optional firewall hardening. The preflight step checks those conditions over SSH before installation starts.

    Host preparation

    Preflight expectations

    • The target host must be RHEL 9 or RHEL 10.
    • The target host must have the microshift and openshift-clients RPMs available through configured repositories.
    • The SSH user must have non-interactive sudo.
    • The plugin will write /etc/crio/openshift-pull-secret and /etc/microshift/config.yaml on the target host.

    Core MicroShift configuration

    If left empty, the target host's current hostname is used for the external kubeconfig path and serving certificates.
    Enter one DNS name or IP per line when external API access needs additional certificate names.
